Installing LTR

Assuming you already have downloaded LTR to you computer, LTR's root directory is supposed to contain CMakeLists.txt file and the following folders: cmake, contrib, data, doc, ltr and ltr_client.

Installing Boost

The simplest way to install boost under Windows is to use an installer from http://www.boostpro.com/download. Unix users can read the official tutorial from http://www.boost.org/doc/libs/1_51_0/more/getting_started/unix-variants.html.

Now LTR uses only multithreaded and multithreaded debug versions of boost, but it is recommended to install DLL variants too.

Installing CMake

Install CMake from www.cmake.org/cmake/resources/software.html and add CMake to the system PATH.

Configuring LTR

Create a directory for building, the default name of which is <LTR_root>/build. Further shell commands will be written as if you've chosen the default <LTR_root>/build directory for building. From the chosen directory run the following shell command:

cmake ../

Here ../ is a relative path to <LTR_root>. This command will configure LTR and generate makefiles or project files, depending on your system and IDE installed. You can specify what to generate manually, as here:

cmake ../ -G "Visual Studio 9 2008"

For the list of all supported generators, simply write

cmake

If you are an LTR's developer, it might be useful to have a shell-script file like the following (example for windows): 

del CMakeCache.txt
cmake ../ -G "Visual Studio 9 2008"
pause

Building LTR

As you already have created your favorite IDE's project, compiling will now be simple. (For example, in Visual Studio one just has to build an ALL_BUILD project.)

After building you can run unittests from <LTR_root>/build. In Visual Studio it can be done, by setting ltr_test as a startup project and pressing Ctrl+F5.

It is important that after successful build you should build an INSTALL project, or (depending on your system and IDE) run

make install

This will create an <LTR_root>/lib directory with 3 static libraries in there: ltr, logog and puppy (with "_d" suffix if the debug build has been used). If you want to use LTR in your project, you have to link to these libraries.

Linking to LTR

Lets start from a rather simple test project, that uses LTR. Here is an example of the test project's CMakeLists.txt:

00001 PROJECT(LTRTestProject CXX C)
00002 CMAKE_MINIMUM_REQUIRED(VERSION 2.6)
00003 
00004 SET(Test_Source_Path ${CMAKE_CURRENT_SOURCE_DIR})
00005 SET(LTR_Source_Path ${Test_Source_Path}/..)
00006 
00007 INCLUDE(${LTR_Source_Path}/cmake/setup_ltr.cmake)
00008 SetUpLTR(${LTR_Source_Path})
00009 
00010 INCLUDE_DIRECTORIES(${Test_Source_Path})
00011 SET(Test_SOURCES ${Test_Source_Path}/file1.cpp
00012                  ${Test_Source_Path}/file2.cpp)
00013 
00014 ADD_EXECUTABLE(test_executable ${Test_SOURCES})
00015 TARGET_LINK_LIBRARIES(test_executable ${LTR_libs})

Here the Test_Source_Path variable is the path to the sources of the test project. Usually it equals to the CMAKE_CURRENT_SOURCE_DIR variable, i.e., the path to the current CMakeLists.txt file.

Then the LTR_Source_Path is specified as a relative path from the Test_Source_Path. Absolute path can also be used:

SET(LTR_Source_Path "C:/Data/Ltr1")

Strings no. 7-8 include a cmake file, which specifies a SetUpLTR macro and run it with LTR_Source_Path parameter. This will set all the necessary variables for using LTR (and also Boost).

Strings no. 10-12 specify a directory for header files of the test project and list test's source files.

Finally, the ADD_EXECUTABLE command creates an executable project called test_executable from sources listed in the Test_SOURCES variable. Then, the TARGET_LINK_LIBRARIES command makes test_executable project use LTR by invoking LTR_libs variable, defined in SetUpLTR macro call.
